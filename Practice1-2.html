JavaScript

  1. JavaScript는 사용자와 상호작용하는 언어이다.
  2. Web browser는 한번 화면에 출력되면 자신을 바꿀 수 없다.
  3. 그러므로, JavaScript는 HTML을 제어하는 언어이다.
- input type="checkbox" : 체크하고 지울 수 있는 박스
- input type="button" value="night" : 버튼을 만들고, 글자 속성은 night

    1. Script 태그

    : script(html코드)로 감싸져 있는 텍스트가 JS다라고 웹브라우저에게 알리는 구분자.
    1. document.write(); : 웹페이지에 어떤 글씨를 출력할 때.
      2. ex. document.write(1+1); 2가 출력됨.
      ex. document.write('hello world'); hello world 글씨 출력.

    2. alert("가나다"); : "가나다"라는 알림 뜸.

    2. Event

    • onclick
    • onchange
    • onkeydown
    1. javascript keydown event attribute ; onkeydown
      2. : 키를 눌렀을 때(keydown), 이벤트를 발생시키는 속성
      ex. input type="text" onkeydown="alert('누름')" : 텍스트 상자 안에서 키를 눌렀을 때, '누름'이라는 알림 발생.

    2. onclick "가나다" : onclick이라는 속성의 값으로는 javascript가 필요함.
      3. onclick 속성의 값으로 자바스크립트를 넣어주면, 사용자가 onclick 속성이 위치한 버튼을 클릭했을 때, javascript 코드 속성값 "가나다"가 실행됨.
      ex. input type="button" value="hi" onclick="alert('bye')" : 자바스크립트 코드인 alert을 실행
      이렇듯, 웹브라우저 위에서 일어나는 일들을 'event'라고 한다. 즉, 어떠한 event가 발생했을 떄, 명령된 js실행하는 것이 onclick.

    3. input type="text" onchange="alert('바뀜')" : 글씨 입력 창. 내용이 변했을 때(라는 이벤트)에서 알림.

    ex. alert ('가나다'.lengh) : 가나다의 문자 개수를 알림으로

    3. 변수와 대입연산자

    ex. x = 1, y=3
    x,y는 변수(variable) ; 1과 같은 상수(constant)는 항상 같은 값을 가지므로 변수 X.
    '='는 Assignment Operator대입연산자. 즉, 오른쪽 항에 있는 값을 왼쪽의 변수에 대입한다 (대입연산자는 좌항과 우항을 결합해서 우항의 값을 만들어냄)

    ex. var name ='jina'
    alert("가나"+name+"다라마"+name+"바사아자차가타파하") ; 변수인 name을 'jina'로 설정.
    코드를 변경할 시, 본문 신경할 것 없이 var name ='jina'만을 수정.

    복합대입연산자 : 위에 있는 코드와 아래의 코드가 같은 코드가 됨.
    ex. var integer = 10
    integer = integer + 10 // integer = 20;
    integer += 10 // integer = integer + 10 = 30
    integer -= 10 // integer = integer - 10 = 20
    integer /= 10 // integer = integer / 10 = 2
    integer *= 10 // integer = integer * 10 = 20

    1. Style Attribute

    특정 태그를 CSS로 디자인하는 방법 : style이라는 속성(html의 문법)내 CSS의 속성을 사용.

    2. Style Tag

    : Style로 감싸져 있는 텍스트가 CSS다라고 웹브라우저에게 알리는 구분자.
    - 웹페이지에서 class 값이 js 인 모든 태그에 대해서 폰 font-weight를 bold로 하겠다.
    예제) 굵은 글씨하고 싶은 단어들에 span class="가나다" 라는 태그를 붙이고, head내 style 태그 안에 아래의 CSS입력
    -> span{font-weight: bold;} : 태그이름이 span인 것에
    -> .가나다{font-weight: bold;} : class값이 '가나다'인 것에 {}내 명령
    -> #라마바{font-weight: bold;} : id값이 '라마바'인 것에 {}내 명령
    -> 우선순위 : 태그> class> id (한 페이지에 중복되는 id는 불가)
    ex. ol class="a"
    style 태그에 ol.a{list-style-type: lower-alpha;} : ol태그 중 class값이 'a'인 것에

    3. Selector 선택자

    : JS/CSS를 통해 어떤 정보를 제어하고 싶을 때, 여러개의 태그를 선택하는 방법
    - 태그 선택자/ id 선택자/ class 선택자
    - div(줄바꿈), span(줄바꿈x)

    5. Object

    ex. var 객체 X = {key값 : value값}

    document.write(key + object[key])
    -> key : 객체 내 key값
    -> 객체.key : key값에 해당하는 데이터(value)

      this.value = 'night';
      }

      < UI vs. API >

      User Interface : 사용자가 시스템을 제어하기 위해 사용하는 조작장치
      ex. Click button

      Application Programming Interface : Web Application을 만들기 위해서 프로그래밍할때 사용하는 조작장치
      ex. onclik = "alert" 에서 alert은 웹브라우저 만드는 사람이 경고창을 실행하는 alert이란 함수를 만들어 논것.
      웹브라우저가 가지고 있는 경고창 기능을 alert이라는 JS의 문법에 따라 실행된다.
      모든 어플리케이션은 API를 프로그래밍적으로(순서대로) 실행하는 것을 통해 만들어진다.

      유용한 property/method의 객체 검색어

    • document : 웹페이지에 있는 어떤 태그를 삭제하고 싶거나, 어떤 태그에 자식객체를 추가하고 싶을 때
    • DOM (Document Object Model) : Document로 부족할 때
    • window : 웹페이지가 아니라 웹브라우저 자체를 제어해야 할 때
    • ex. 현재 열려있는 웹페이지의 주소를 알아내거나 새창을 열 때
    • ex. 웹브라우저의 화면크기를 JS를 통해 알아낼 때
    • ajax : 웹페이지를 리로드하지 않고 정보를 변경하고 싶을 때
    • cookie : 웹페이지가 리로드 되어도 현재 상태를 유지하고 싶을 때 (사용자를 위한 개인화된 서비스 이용가능)
    • offline web applicaion : 인터넷이 끊겨도 동작하는 웹페이지 만들 때
    • webRTC : 화상통신 WebApp
    • speech로 시작하는 API들 : 사용자의 음성을 인식하고, 음성으로 정보 전달
    • webGL : 3차원 그래픽으로 게임같은 거 만들 때
    • webVR : 가상현실
